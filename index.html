Three Questions

1. What makes object oriented design integral to how you think about coding/programming/computer science?

I conceptualize objeects as a sort of data type in programming. They become useful especially when the code is especially large and contains many similar groups of variables. These groups of variables often describe an identical property but are different values depending on what is being described. As an example, people can be described using identical variables, all having different values. Names will be names but be different on the person, eye color, race, height, gender and so on. Objects help programmers like myself simplify this repetitiveness with classes and objects.

There are four main aspects of object oriented design: abstraction, encapsulation, inheritance and polymorphism. All of these concepts help me think more critically and abstracly while programming. The first is abstraction. This concept helps me think broader while trying to simulate an object. Instead of saying this container is 50 centimeters tall, I now thing: this container is some height. This adds flexibility to my code. Next is encapsulation. With this idea, I now group related variables together to create a blueprint that is easier to think about. This helps with thinking critically. The next two are more abstract. Inheritance is the idea that most things come from a same idea, but have their own modifications. While programming, I think about how an object can be further grouped in its own category. In essence, it is the creation of a hierarchy of objects. Polymorphism is the final concept that strengthens how I think of programming in an object oriented manner, it provides me with the ability to think of how an object can be unique even within its group. These ideas together create a way of thinking more abstractly and solve problems with the knowledge that I already have from other problems. Every problem is from the same set as another that has already been solved.
